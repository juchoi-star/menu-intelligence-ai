# PRD — Menu Intelligence AI

## 1. 배경 / 문제

외식 프랜차이즈(생전포차, 11개 가맹점) 메뉴개발팀은 매월 POS에서 "메뉴별 매출 순위
집계" 엑셀을 추출하지만, 병합셀·소계·총계가 섞인 raw 파일을 수작업으로 비교하느라
- 어떤 메뉴가 실제로 성장/감소했는지,
- 성장이 **할인 때문인지 상품력 때문인지**,
- 순위·기여도가 어떻게 바뀌었는지
를 빠르게 판단하기 어렵다.

## 2. 목표

**월별 POS 파일 2개(전월·당월)를 업로드하면 AI가 자동으로 비교 분석하고
스토리형 인사이트와 PDF 보고서를 생성한다.**

### 성공 기준 (MVP)
- [x] 두 엑셀 업로드 → 자동 파싱(병합셀/소계/총계 처리) → 비교 분석
- [x] 파서 합계가 POS 총계블록과 정확히 일치
- [x] 대시보드 5개 KPI + 분류/기여도 차트
- [x] 메뉴 인사이트 9종(상승/하락/기여/주문/순위±/신규/중단/할인없이성장)
- [x] 메뉴 상세 + 가맹점별 증감표
- [x] 가맹점 TOP20 분석
- [x] AI 스토리 내러티브 + 추천 액션
- [x] 7단 목차 PDF 보고서

## 3. 사용자 / 페르소나

- **메뉴개발 담당자**: 신메뉴 안착·기존메뉴 개편 판단. "할인 없이 성장한 메뉴"에 관심.
- **운영 관리자**: 가맹점별 편차·이익률 모니터링.

## 4. 핵심 사용자 흐름

1. 대시보드에서 전월/당월 엑셀 업로드 → 분석 실행
2. KPI·차트로 전체 흐름 파악
3. 메뉴 분석 → 관심 메뉴 클릭 → 상세(가맹점별 증감) 확인
4. AI Report에서 스토리 확인 및 PDF 다운로드 → 회의 자료로 사용

## 5. 기능 요구사항

| # | 기능 | 우선순위 |
|---|------|----------|
| F1 | POS 엑셀 파싱(병합/소계/총계/결손행) | P0 |
| F2 | 전월/당월 자동 판별·비교 분석 | P0 |
| F3 | 대시보드 KPI/차트 | P0 |
| F4 | 메뉴 인사이트 9종 | P0 |
| F5 | 메뉴 상세 + 가맹점별 증감 | P0 |
| F6 | 가맹점 분석 TOP20 | P1 |
| F7 | AI 내러티브(GPT/규칙) | P0 |
| F8 | PDF 보고서 | P1 |
| F9 | 외부요인(Weather/News) | P2 (예정) |

## 6. 비기능 요구사항

- OpenAI 키/외부 DB 없이도 **로컬에서 즉시 실행**(SQLite + 규칙 기반 폴백).
- 파일 포맷 고정 가정, 단 헤더행/기간은 동적 탐지로 견고성 확보.
- 유지보수 용이: 도메인 로직(core)·API·DB(Repository) 분리, 타입힌트/Pydantic.

## 7. 범위 제외 (MVP)

- 실시간 POS 연동, 3개월+ 시계열, 사용자 인증/권한, 원가 기반 정밀 이익률.
