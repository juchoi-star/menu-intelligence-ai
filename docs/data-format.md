# POS "메뉴별 매출 순위 집계" 데이터 포맷

매월 POS에서 추출되는 엑셀은 항상 동일한 레이아웃을 가진다.
파서(`backend/app/core/parser.py`)는 이 포맷을 기준으로 구현되어 있으며,
실제 5월/6월 파일에서 **총계 블록과 100% reconcile** 검증을 마쳤다.

## 시트 레이아웃

| 행 | 내용 |
|----|------|
| 1 | 제목: `메뉴별 매출 순위 집계` |
| 3 | 조회 범위: 예) `전체 가맹점` |
| 4 | `조회일자 : YYYY-MM-DD ~ YYYY-MM-DD` → **기간 추출** |
| 5 | 출력시간 |
| 7~8 | 2단 병합 헤더 |
| 9~ | 데이터 |

## 데이터 계층 구조

```
가맹점 (B:가맹점코드, C:가맹점명)
  └─ 메뉴분류 (D:메뉴분류)
       └─ 메뉴 (E:메뉴코드, F:메뉴명)  ← 실제 판매 데이터
       └─ ... 
     [분류 Total 행]        ← E == 'Total'
  [가맹점 소계 행]           ← D == '소계'
  [중복 롤업 행]             ← K(비율/가맹점) == '200%'
...
[총계 블록]                 ← A(No)가 텍스트('총 주문건수' 등)
```

- 가맹점코드/명, 메뉴분류는 **그룹의 첫 행에만** 표기 → **Forward-Fill** 필요.
- 가맹점이 바뀌면 분류 컨텍스트를 리셋한다.

## 주요 컬럼 매핑

| 열 | 필드 | 열 | 필드 |
|----|------|----|------|
| A | No | Q | 실매출액(합계) |
| B | 가맹점코드 | T | 순매출 |
| C | 가맹점명 | U | 부가세 |
| D | 메뉴분류 | V | 매출원가(합계) |
| E | 메뉴코드 | Y | 매출이익(합계) |
| F | 메뉴명 | AB | 기타금액 |
| G | 메뉴단가 | AC | 총판매금액(합계) |
| I | 주문건수(합계) | K | 비율(가맹점) |
| M | 주문금액(합계) | L | 폐기건수 |
| N | 할인금액(합계) | H | 중량 |

> 이 데이터셋은 매출원가(V)가 0으로 채워져 있어, 이익률(매출이익/실매출)이
> VAT를 제외한 약 90.9% 수준으로 계산된다. 원가 데이터가 채워지면 자동으로 정밀해진다.

## 집계행 제거 규칙 (핵심)

분석 시 **Total / 소계 / 총계 / 중복 롤업** 행은 모두 제거한다.
"집계행 제외" 방식으로 판별하여, 아래 조건 중 하나라도 해당하면 제외:

1. `No(A)`가 정수 시퀀스가 아님 → 파일 끝 **총계 블록**
2. `메뉴코드(E) == 'Total'` → **분류 소계**
3. `메뉴분류(D) == '소계'` → **가맹점 소계**
4. `비율/가맹점(K)`에 `'200%'` 포함 → **중복 롤업 행**

### ⚠️ 결손 행 보존
POS 데이터 결손으로 **메뉴코드/명이 비어 있어도 판매(주문건수·실매출)가 있는 실제 행**이 존재한다
(예: 5월 R1391 — 8,000원 × 10건). 이름 기준으로 버리면 매출이 누락되므로,
이런 행은 플레이스홀더 코드/명(`(미상 메뉴)`)을 부여해 **보존**한다.
→ 이렇게 해야 파서 합계가 POS 총계블록과 정확히 일치한다.

## 그룹 분류 (주류 / 음식 / 기타)

메뉴개발팀 관점에서 주류와 음식은 매출 동학이 달라 **상위 그룹으로 분리 분석**한다
(`backend/app/core/groups.py`). 미등록 분류는 안전하게 `기타`로 폴백한다.

| 그룹 | 포함 분류 |
|------|-----------|
| **주류** | 주류, 막걸리메뉴 |
| **음식** | 전, 탕&식사, 간단안주, 세트, 튀김 |
| **기타** | 음료(비주류), 이벤트, 기타, 직원호출 |

분리 분석의 핵심:
- **순위(rank)** 와 **기여도(contribution_pct)** 는 **그룹 내**에서 계산한다
  (참이슬과 김치전을 한 순위표에 두지 않음). 전체 대비 기여도는 `contribution_overall_pct` 로 별도 제공.
- 상승/하락/신규/순위변화 등 인사이트는 그룹별(`groups[]`)로 각각 산출.
- 가맹점 분석은 매출을 주류/음식/기타로 분해(`group_sales_curr`).

## 계산 지표 정의

| 지표 | 정의 |
|------|------|
| 성장률(%) | (당월 − 전월) / 전월 × 100 (전월 0이면 신규) |
| 할인율(%) | 할인금액 / 주문금액 × 100 |
| 이익률(%) | 매출이익 / 실매출 × 100 |
| 순위 | 실매출 내림차순 (월별) |
| 순위변화 | 전월순위 − 당월순위 (+면 상승) |
| 기여도(%) | 당월 실매출 / 당월 전체 실매출 × 100 |
| 할인없이 성장 | 성장률 > 0 AND 할인율 증가 ≤ 0 AND 당월매출 ≥ 5만 |
